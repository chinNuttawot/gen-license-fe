# 📝 การทำงานของหน้าเว็บ Frontend (UI Flow)

เอกสารนี้อธิบายการทำงานของฝั่ง Frontend (React / Next.js) ในการสร้างและดาวน์โหลด License 

---

## 🎨 สถาปัตยกรรมหน้าจอ (Component Structure)
หน้าจอหลักถูกแบ่งออกเป็น 2 ส่วนการทำงาน (ผ่าน `app/page.tsx`) เพื่อให้ผู้ใช้งานมองเห็นข้อมูลที่กรอก และผลลัพธ์ได้อย่างชัดเจน:

1. **`LicenseForm.tsx` (แบบฟอร์มฝั่งซ้าย)**
   - ทำหน้าที่รับข้อมูลจากผู้ใช้ เช่น ชื่อบริษัท, ประเภท License, จำนวน/HWID และวันหมดอายุ
   - มีระบบ Validate ข้อมูลเบื้องต้น (เช่น วันหมดอายุต้องเป็นอนาคต, ถ้าเป็น Station-based ต้องกรอก HWID ให้ครบ)
   - เมื่อกดปุ่ม **"Generate Token"** จะส่งข้อมูลไปที่ API เพื่อขอดูผลลัพธ์ (Preview)
   - มีปุ่ม **"Reset"** สำหรับล้างค่าแบบฟอร์มทั้งหมด

2. **`TokenDisplay.tsx` (ผลลัพธ์ฝั่งขวา)**
   - ทำหน้าที่แสดง Token ที่ถูกสร้างขึ้นมาจากฝั่ง Backend
   - หากมีการ Generate สำเร็จ จะขึ้นแสดงจำนวน Token พร้อมปุ่ม Copy และ Export
   - **ระบบจำกัดการมองเห็น (Performance):** หาก Token มีจำนวนมากเกินไป ระบบจะพับเก็บ (Collapse) และโชว์เฉพาะส่วนแรกสุด เพื่อไม่ให้หน้าเว็บค้าง

---

## 🔄 ลำดับการทำงาน (User Flow)

### 1. การสร้างและดูตัวอย่าง (Generate & Preview)
เมื่อผู้ใช้งานกรอกข้อมูลและกดปุ่ม **Generate**:
- หน้าจอจะเรียกใช้ `licenseService.generate(...)` ไปที่ Backend (`POST /api/license/generate`)
- หาก Backend ทำงานสำเร็จ หน้าจอฝั่งขวา (`TokenDisplay`) จะแสดงรายการ Token ทั้งหมดให้ผู้ใช้ตรวจสอบ
- **ในขั้นตอนนี้ ยังไม่มีการบันทึกข้อมูลใดๆ ลงฐานข้อมูลจริง**

### 2. การดาวน์โหลดและบันทึก (Export & Save)
เมื่อผู้ใช้งานพึงพอใจกับ Token ที่เห็น และกดปุ่ม **"Export .aglic"**:
- หน้าจอจะส่ง Token และ Metadata ของชุดนั้นๆ ไปที่ Backend ผ่าน `licenseService.exportBundle(...)`
- Backend จะทำการเข้ารหัสและเซฟข้อมูลลง Database 
- เบราว์เซอร์จะทำการดาวน์โหลดไฟล์ `.aglic` เข้าเครื่องผู้ใช้
- **ระบบป้องกันการกดซ้ำซ้อน (Anti-Spam UI):** ทันทีที่การ Export สำเร็จ ปุ่ม Export จะเปลี่ยนสถานะเป็น **"Exported!" (สีเขียว)** และถูก **Disable ถาวร** ป้องกันไม่ให้ผู้ใช้งานเผลอกดโหลดรัวๆ
- หากต้องการ Export ใหม่ ผู้ใช้จะต้องไปกด Generate สร้าง Token ชุดใหม่ก่อนเท่านั้น

### 3. การล้างข้อมูล (Reset State)
เมื่อผู้ใช้งานกดปุ่ม **"Reset"** ที่แบบฟอร์ม:
- ค่า State ในฟอร์มทั้งหมดจะถูกตั้งค่ากลับเป็นค่าว่าง
- ส่งผลลัพธ์ไปล้าง State (`tokens`, `meta`, `error`) บน `page.tsx` ด้วย ทำให้หน้าจอฝั่งขวากลับไปสู่สถานะเริ่มต้น "No tokens yet" ทันที
